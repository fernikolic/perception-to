# Update Frequency

> How often data refreshes in Perception

## Overview

Perception maintains multiple data pipelines that update at different frequencies, ensuring you have access to both real-time coverage and comprehensive historical analysis.

## Update Schedules

### Data Collection

| Source Type | Update Frequency |
|-------------|------------------|
| **News Sources** | Multiple times daily |
| **Social Media** | Near real-time |
| **Wire Services** | Continuous |
| **Weekly Publications** | As published |

### Data Processing

| Process | Frequency |
|---------|-----------|
| **Sentiment Analysis** | On ingestion |
| **Topic Extraction** | On ingestion |
| **Search Indexing** | Within minutes |
| **Trend Analysis** | Real-time |

### User-Facing Features

| Feature | Update Frequency |
|---------|------------------|
| **Intelligence Feed** | Real-time (on page load) |
| **Trends** | Real-time |

## Alert Timing

### Real-Time Alerts
- **Check frequency**: Every 30 minutes
- **Delivery**: Within 30 minutes of qualifying coverage
- **Rate limiting**: Maximum 5 per day, 2-hour gap between same-keyword alerts

### Weekly Digest
- **Generation**: Weekly on your chosen day
- **Delivery time**: Your selected hour in your timezone
- **Data included**: Previous 7 days

---

## Trend Analysis

### How Trends Update

1. **Real-time extraction**: Our proprietary systems analyze the most recent data points
2. **Pattern matching**: Identifies new and continuing narratives
3. **Deduplication**: Merges similar trends to avoid repetition
4. **Signal strength**: Updates based on coverage volume

### Trend Freshness

| Signal Strength | Typical Age |
|-----------------|-------------|
| **Early** | Breaking (last few hours) |
| **Emerging** | Recent (last 24-48 hours) |
| **Strong** | Established (may be ongoing for days) |

## What Affects Freshness

### Factors in Data Timing

1. **Source publication time**: When the outlet publishes
2. **Collection time**: When our systems gather the content
3. **Processing time**: Sentiment analysis and indexing
4. **Display time**: When you view in the app

### Typical End-to-End Timing

| Source Type | Publication → Searchable |
|-------------|-------------------------|
| **Wire Services** | 5-15 minutes |
| **Major News Sites** | 15-30 minutes |
| **Crypto Publications** | 15-30 minutes |
| **Social Media** | 5-15 minutes |

---

## Tips for Time-Sensitive Monitoring

1. **Use real-time alerts** for breaking coverage
2. **Check Trends frequently** during volatile markets
3. **Refresh the feed** to see latest data points
4. **Set appropriate date filters** when searching
