# SERP Analysis Prompt Templates

## Quick SERP Check

```
Analyze SERP for: "[KEYWORD]"

Use WebSearch to find current results, then analyze:

1. **Top 10 Results:**
   | Position | Domain | Title | Type |
   |----------|--------|-------|------|

2. **Intent Analysis:**
   - Informational / Transactional / Navigational?
   - What content type dominates?
   - Featured snippets present?

3. **Perception's Position:**
   - Do we appear? Where?
   - If not, what's blocking us?

4. **Quick Wins:**
   - Title optimization opportunities
   - Content gaps vs top results
   - Structured data advantages
```

## Deep SERP Analysis

```
Perform deep SERP analysis for: "[KEYWORD]"

**Phase 1: Gather Data**
1. Search the keyword with WebSearch
2. Fetch top 5 results with WebFetch (if possible)
3. Note SERP features (featured snippets, FAQs, etc.)

**Phase 2: Analyze Results**
For each top 5 result:
- URL and domain authority estimate
- Title tag (exact text, character count)
- Meta description (if visible)
- Content type and format
- Key topics covered
- Word count estimate
- Structured data visible

**Phase 3: Identify Patterns**
- Common title formulas
- Recurring topics/subtopics
- Content depth required
- Media/image usage
- Internal linking patterns

**Phase 4: Build Strategy**
- Content angle to differentiate
- Title/description approach
- Required topics to cover
- Optimal content length
- Structured data to implement
- Internal linking plan

**Output:**
- SERP landscape summary
- Top competitor analysis
- Content brief for ranking
- Specific recommendations
```

## Featured Snippet Targeting

```
Analyze featured snippet opportunity for: "[KEYWORD]"

1. **Current State:**
   - Is there a featured snippet?
   - What type? (paragraph, list, table)
   - Who owns it currently?
   - What's the source content?

2. **Snippet Optimization:**
   - Craft concise answer (40-60 words for paragraph)
   - Format for snippet type
   - Include the query in the answer
   - Provide clear, authoritative response

3. **Page Requirements:**
   - Question as H2 heading
   - Direct answer immediately after
   - Supporting content below
   - Structured data support

4. **Output:**
   - Recommended snippet content
   - Page structure for targeting
   - Implementation notes
```

## "People Also Ask" Analysis

```
Analyze "People Also Ask" for: "[KEYWORD]"

1. **Capture PAA Questions:**
   List all questions that appear in PAA box

2. **Categorize Questions:**
   - What is...? (definitional)
   - How to...? (instructional)
   - Why...? (explanatory)
   - Best/Top...? (listicle)
   - Comparison...? (versus)

3. **Content Opportunities:**
   - Which questions can we answer?
   - Where should answers live on our site?
   - FAQ schema opportunities
   - New content ideas

4. **Output:**
   - PAA question list
   - Answer drafts for each
   - Page placement recommendations
   - Schema markup suggestions
```

## Voice Search Optimization

```
Optimize for voice search queries:

**Voice Search Characteristics:**
- Longer, conversational queries
- Question format
- Local intent common
- Featured snippet dominant

**Target Queries:**
- "What is the best crypto media monitoring tool?"
- "How do I track bitcoin news coverage?"
- "What's the difference between Perception and LunarCrush?"

**Optimization Strategy:**
1. Create FAQ content targeting these queries
2. Use natural, conversational language
3. Provide direct, concise answers
4. Implement FAQPage schema
5. Optimize for featured snippets

**Output:**
- Voice search query targets
- Content optimizations
- FAQ additions
- Schema implementation
```
